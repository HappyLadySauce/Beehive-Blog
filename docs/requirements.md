# 个人博客系统需求文档

## 1. 项目概述

### 1.1 项目名称
Beehive-Blog 个人博客系统

### 1.2 项目背景
本项目是一个基于 Go 语言开发的个人博客系统，旨在提供一个功能完善、易于管理的博客平台，支持文章发布、标签管理、评论互动、附件管理等核心功能。

### 1.3 技术栈
- **后端语言**: Go
- **数据库**: PostgreSQL
- **缓存**: Redis
- **API 文档**: Swagger
- **部署**: Docker

## 2. 用户角色定义

### 2.1 角色说明

| 角色 | 角色标识 | 权限级别 | 说明 |
|------|----------|----------|------|
| 访客 | `guest` | 0 | 访客用户，可浏览内容和评论 |

| `updated_at` | datetime | 更新时间 |

## 4. 功能需求

### 4.1 用户认证模块

#### 4.1.1 用户注册
- **功能描述**: 新用户注册账号
- **输入**: 用户名、密码、邮箱（可选）
- **输出**: 注册成功/失败信息
- **业务规则**:
  - 用户名必填，长度 3-20 字符
  - 密码长度 6-32 字符，需加密存储
  - 邮箱格式校验（如提供邮箱，需符合标准邮箱格式）
  - 需要邮箱验证，验证链接发送至用户邮箱，用户需点击链接激活账号
  - 验证链接有效期 24 小时

#### 4.1.2 用户登录
- **功能描述**: 用户身份验证
- **输入**: 用户名/邮箱、密码
- **输出**: JWT Token、用户信息
- **业务规则**:
  - 支持用户名或邮箱登录
  - 密码错误次数限制（5次/小时）
  - Token 有效期 7×24 小时

#### 4.1.3 密码找回
- **功能描述**: 忘记密码后重置密码
- **输入**: 邮箱
- **业务规则**:
  - 发送重置链接到用户邮箱
  - 重置链接有效期 1 小时
  - 重置成功后旧 Token 失效

### 4.2 文章管理模块

#### 4.2.1 查看文章列表
- **功能描述**: 浏览文章列表，支持多维度筛选
- **权限**: 所有用户
- **接口**: `GET /articles`
- **查询参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `page` | int | 否 | 页码，默认 1 |
| `pageSize` | int | 否 | 每页条数，默认 10 |
| `keyword` | string | 否 | 关键字搜索（标题、内容） |
| `category` | string | 否 | 分类标识，如 `category-dycrduq4` |
| `tag` | string | 否 | 标签标识，如 `tag-6k3rxgup` |
| `status` | string | 否 | 文章状态，默认 `published` |
| `author` | string | 否 | 作者用户名 |
| `sort` | string | 否 | 排序方式：`newest`/`oldest`/`popular`，默认 `newest` |

- **业务规则**:
  - `category` 和 `tag` 可同时筛选，结果为交集
  - 仅返回状态为 `published` 的文章（管理员可查看全部）
  - 支持多标签筛选（逗号分隔）

#### 4.2.2 查看文章详情
- **功能描述**: 查看单篇文章详情
- **权限**: 所有用户
- **接口**: `GET /articles/:id`
- **响应内容**:
  - 文章基本信息（标题、内容、作者、发布时间等）
  - 分类信息
  - 标签列表
  - 点赞数、收藏数、评论数
  - 上一篇/下一篇导航
- **业务规则**:
  - 阅读量 +1（需防刷机制）
  - 私密文章仅管理员可见

#### 4.2.3 增加文章（管理员）
- **功能描述**: 发布新文章
- **权限**: 仅管理员
- **接口**: `POST /articles`
- **输入**: 标题、内容、分类（可选）、标签（可选）、封面图（可选）、状态
- **业务规则**:
  - 标题必填，长度 1-200 字符
  - 内容支持 Markdown 格式
  - 支持草稿和发布两种状态

#### 4.2.4 编辑文章（管理员）
- **功能描述**: 修改已有文章
- **权限**: 仅管理员
- **接口**: `PUT /articles/:id`
- **业务规则**:
  - 保留文章历史版本（可选）
  - 更新时间自动记录

#### 4.2.5 删除文章（管理员）
- **功能描述**: 删除文章
- **权限**: 仅管理员
- **接口**: `DELETE /articles/:id`
- **业务规则**:
  - 软删除，保留数据可恢复
  - 删除时关联删除文章-标签关系

#### 4.2.6 文章分类管理（管理员）
- **功能描述**: 管理文章分类
- **权限**: 仅管理员
- **功能点**:
  - 创建分类 `POST /categories`
  - 编辑分类 `PUT /categories/:id`
  - 删除分类 `DELETE /categories/:id`（需检查是否有关联文章）
  - 查看分类列表 `GET /categories`

#### 4.2.7 文章状态管理（管理员）
- **功能描述**: 设置文章状态
- **权限**: 仅管理员
- **接口**: `PUT /articles/:id/status`
- **状态定义**: 见 [3.2 文章状态](#32-文章状态)

### 4.3 标签管理模块

#### 4.3.1 查看标签列表
- **功能描述**: 浏览所有标签
- **权限**: 所有用户
- **接口**: `GET /tags`
- **查询参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `page` | int | 否 | 页码，默认 1 |
| `pageSize` | int | 否 | 每页条数，默认 50 |
| `sort` | string | 否 | 排序方式：`name`/`count`/`newest`，默认 `name` |

- **响应内容**:
  - 标签ID、名称、颜色
  - 关联文章数量
  - 创建时间

#### 4.3.2 查看标签详情
- **功能描述**: 查看单个标签详情及关联文章
- **权限**: 所有用户
- **接口**: `GET /tags/:id`
- **响应内容**:
  - 标签基本信息
  - 关联文章列表（分页）
  - 相关标签推荐

#### 4.3.3 按标签筛选文章
- **功能描述**: 根据标签筛选文章列表
- **权限**: 所有用户
- **接口**: `GET /articles?tag={tagId}`
- **查询参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `tag` | string | **是** | 标签标识，支持多标签（逗号分隔） |
| `page` | int | 否 | 页码，默认 1 |
| `pageSize` | int | 否 | 每页条数，默认 10 |
| `sort` | string | 否 | 排序方式：`newest`/`oldest`/`popular`，默认 `newest` |

- **业务规则**:
  - 单标签筛选：`/articles?tag=tag-6k3rxgup`
  - 多标签筛选（交集）：`/articles?tag=tag-6k3rxgup,tag-abc123`
  - 仅返回已发布文章
  - 结果按发布时间倒序排列

#### 4.3.4 增加标签（管理员）
- **功能描述**: 创建新标签
- **权限**: 仅管理员
- **接口**: `POST /tags`
- **输入**: 标签名称、颜色（可选）、描述（可选）
- **业务规则**:
  - 标签名称唯一，长度 1-50 字符
  - 颜色格式：十六进制色值，如 `#FF5733`

#### 4.3.5 编辑标签（管理员）
- **功能描述**: 修改标签信息
- **权限**: 仅管理员
- **接口**: `PUT /tags/:id`
- **可修改字段**:
  - 标签名称
  - 颜色
  - 描述
  - 排序权重

#### 4.3.6 删除标签（管理员）
- **功能描述**: 删除标签
- **权限**: 仅管理员
- **接口**: `DELETE /tags/:id`
- **业务规则**:
  - 删除时自动解除与文章的关联关系
  - 有关联文章的标签需二次确认

#### 4.3.7 标签云
- **功能描述**: 获取标签云数据
- **权限**: 所有用户
- **接口**: `GET /tags/cloud`
- **查询参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `limit` | int | 否 | 返回数量，默认 50，最大 100 |

- **响应内容**:
  - 标签名称、颜色
  - 文章数量（用于计算字体大小）
  - 按文章数量排序

- **权限**: 仅管理员
- **业务规则**:

#### 4.4.4 评论状态管理（管理员）

## 5. 用户等级系统

### 5.1 等级规则

| 等级 | 等级名称 | 所需经验值 | 升级条件 |
|------|----------|------------|----------|
| Lv.1 | 初来乍到 | 0 | 注册即获得 |
| Lv.2 | 博客新手 | 100 | 注册满7天 或 有效阅读10篇文章 |
| Lv.3 | 活跃读者 | 300 | 注册满30天 或 有效阅读30篇文章 |
| Lv.4 | 资深访客 | 600 | 注册满90天 或 有效阅读60篇文章 |
| Lv.5 | 博客达人 | 1000 | 注册满180天 或 有效阅读100篇文章 |
| Lv.6 | 资深博主 | 2000 | 注册满365天 或 有效阅读200篇文章 |

### 5.2 经验值获取规则

| 行为 | 经验值 | 限制条件 |
|------|--------|----------|
| 每日登录 | +5 | 每日限1次 |
| 有效阅读文章 | +10 | 单篇文章限1次，需停留≥3分钟 |
| 发表评论 | +15 | 每日限20条 |
| 文章被点赞 | +2 | 无限制 |
| 注册时间奖励 | 每天+1 | 按注册天数累计 |

### 5.3 有效阅读判定
- **判定条件**: 用户在文章页面停留时间 ≥ 3 分钟
- **防刷机制**:
  - 同一用户同一文章24小时内只计算一次
  - 检测页面可见性（切换标签页不计时）
  - 检测用户行为（滚动、点击等交互）

## 7. API 设计规范

### 7.1 响应格式

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

### 7.2 分页参数

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `page` | int | 1 | 页码 |
| `pageSize` | int | 10 | 每页条数，最大 100 |

### 7.3 错误码定义

| 错误码 | 说明 |
|--------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未授权 |
| 403 | 禁止访问 |
| 404 | 资源不存在 |
| 429 | 请求过于频繁 |
| 500 | 服务器内部错误 |

## 9. 非功能需求

### 9.1 性能需求
- 页面加载时间 < 2 秒
- API 响应时间 < 500ms（P95）
- 支持并发用户 100+

### 9.2 安全需求
- 密码使用 bcrypt 加密
- API 使用 JWT 认证
- 防止 SQL 注入（使用 ORM/参数化查询）
- XSS 防护
- 接口限流（每分钟 100 次）

### 9.3 可用性需求
- 系统可用性 99.9%
- 支持数据定期备份
- 错误日志记录

### 9.4 兼容性需求
- 支持主流浏览器（Chrome, Firefox, Safari, Edge）
- 移动端适配
